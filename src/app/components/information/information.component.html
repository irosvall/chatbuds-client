About

This website is a student project made 2021 first year at Linnaeus University by Ida Rosvall.

The project was made as a challenge for me to learn or get comfortable with new techniques such as Angular and socket.io. My main focus was on the backend because I made the decision to learn and create my own first microservice-architecture, though a small one it was a good learning experience and challenging to make work.

My hope with this project was that it would become a nice environment to meet new friends. By not seeing what each person looks like, where they come from etc. makes it a nice experience to fully dig into new friendships without eyes of prejudice.

Data we store

Data stored peristent at ChatBuds

Data stored peristent, meaning stored for all time till account is deleted or database is shut down or reseted are:

  • Infromation shared during registration: Email, username, password. Though password is hashed and salted meaning it can never be read back and therefore safe.
  • Which users you are friends with.
  • Sent and recieved friend requests.
  • Cookie: The cookie is valid for around 8,2 hours before it's not valid and the user needs to log in again.

Data stored non-peristent at ChatBuds

Data stored non-peristent, meaning stored for a limited amount of time are:

  • Private messages are stored on the client up till 300 messages per friend until the user refreshes the page. This makes it possible to go between friends and chat without messages dissapearing. It is not viewable by other users.
  • Your socket connection is stored non-persistant in the process of searching for a chatting buddy. It is removed when leaving that page or when a chatting buddy is found.

Cookie information

A cookie is used on the website for the server to remember which user is making the requests, to keep the user logged-in for example. Cookies are valid for around 8,2 hours before they are not valid and the user needs to log in again.

The cookie holds nothing stored in itself except an unique identifier that you are the one making the requests. It is not used for marketing or statistics.
